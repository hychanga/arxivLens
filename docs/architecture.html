arXivLens — 系統架構與部署位置

arXiv／HBR 論文聚合平台 · AI 摘要與多語翻譯

產生日期：2026-06-14 · 環境：Production · 來源：repo 設定檔 + 部署紀錄

1. 系統總覽

arXivLens 採 前後端分離 架構：Next.js 前端部署於 Vercel，Spring Boot 後端容器化部署於 Google Cloud Run， 資料庫使用 TiDB Cloud（MySQL 相容）。後端對外整合 arXiv／HBR 內容來源與 Google Gemini（AI 摘要與翻譯）、 Resend（通知信）。定時抓取由 GitHub Actions 以 cron 觸發後端的 token 保護端點完成。

4. CI/CD 與部署流程

backend/** 變更
▶
Cloud Build
build → push → deploy → prune
▶
Cloud Run
新 revision 上線
  • Cloud Build 步驟：docker build（backend/Dockerfile）→ push（Artifact Registry）→ gcloud run deploy → 修剪舊 revision（保留 3）。
  • 部署步驟只傳 --image，因此保留既有 env／Secret 綁定；執行期設定一律以 gcloud run services update 帶外管理，不放進 CI。
  • 前端：推送 main 由 Vercel 自動建置部署。

5. 執行期設定（Cloud Run）
